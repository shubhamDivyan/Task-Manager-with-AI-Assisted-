# Reflection on Using AI Tools During Development

While working on the Task Manager assignment, I used an AI code editor to assist with different parts of the development process. This was an interesting experience because it changed the way I usually approach coding tasks. Instead of writing every line of code manually from scratch, I interacted with the AI by describing what I wanted to build and then reviewing the code it generated. Through this process, I learned that AI tools can be very helpful for speeding up development, but they still require careful review from the developer.

One moment where the AI saved me a significant amount of time was during the initial backend setup. Normally, when creating a Node.js project, I would need to install Express, create a server file, configure middleware such as CORS, and set up JSON request parsing. Instead of doing this manually, I prompted the AI to generate a basic Express server with CORS enabled and JSON support. The AI quickly produced a working server structure with the necessary configuration. This saved time because I did not have to write the common boilerplate setup code myself. The prompt worked well because it clearly described the framework being used and the basic features required in the server.

Another situation where AI was helpful was when generating the CRUD routes for the task manager API. Writing routes for GET, POST, PATCH, and DELETE operations can be repetitive because they follow similar patterns. I asked the AI to create API routes for managing tasks stored in an in-memory array. The AI generated a structured version of the routes with basic logic for creating tasks, updating completion status, and deleting tasks. This helped me move faster in development because I had a working starting point that I could modify and improve.

However, the AI did not always generate perfect code. One issue appeared in the React frontend when rendering the list of tasks. The AI-generated code used the `map()` function to display the tasks but did not include a `key` property for each list item. React requires a unique key when rendering lists to help track updates efficiently. I detected this problem when the browser console showed a warning message about missing keys in list elements. After identifying the issue, I fixed it by adding `key={task.id}` to the list item component.

Another problem occurred in the backend DELETE route. The AI-generated code removed tasks from the array but did not check whether the task actually existed before returning a response. This meant the API could return a success message even when no task was deleted. I noticed this while reviewing the code and thinking about possible edge cases. To improve it, I added a validation check to confirm whether the task existed before sending the response.

Working with an AI editor changed my workflow compared to writing code completely from scratch. Instead of starting with empty files and building everything step by step, I began by describing the functionality I wanted and letting the AI generate an initial version of the code. I then reviewed, tested, and modified that code. This made development faster, especially for repetitive tasks, but it also required careful reading and understanding of the generated code.

I would consider using AI tools in a real project because they can help developers work more efficiently and quickly generate common patterns. However, I would not rely on them completely. Developers still need to review the logic, test the code, and ensure it follows good practices.
